Procyon ARMlib
C-Language Function Library for ARM microcontrollers

Written by Pascal Stang | Updated:

News and Notes

  • ARMlib is still being developed and some files and docs are incomplete. Nonetheless, you may find it useful (I use it daily).
  • ARMlib has recently been updated so that it can be used for a variety of ARM microcontroller families.
    Microcontroller-specific drivers and code are now contained in /armlib/arch/[mcu family]/.
  • If you used ARMlib back when it was only for LPC2000 processors, the new processor family support will require some modification of your old makefiles. See /armlib/examples/ for help on this.
  • Several parts of ARMlib have been ported from AVRlib but have not yet been properly modified to fit the ARM architecture. I'll try to get to these when I can.

Supported processors

  • Atmel AT91 series (developed and tested mostly on AT91SAM7 parts)
  • Philips LPC2000 series (tested on LPC2106, LPC2214, and others)
  • Analog Devices ADuC7000 series (light support at the moment)
